# Тестирование Этапа 2: Авторизация

## Предварительные требования

1. Supabase проект настроен
2. Схема БД применена (supabase_schema.sql)
3. `.env.local` заполнен корректными ключами
4. Зависимости установлены: `npm install`

## Шаги тестирования

### 1. Запуск проекта

```bash
npm run dev
```

Откройте http://localhost:3000

### 2. Проверка главной страницы

- ✅ Отображается логотип NiceTry
- ✅ Видны кнопки "Войти" и "Каталог товаров"
- ✅ Отображаются 3 блока преимуществ
- ✅ Header и Footer на месте

### 3. Тестирование регистрации/входа

1. Нажмите "Войти" в Header
2. Введите email (например, test@example.com)
3. Нажмите "Получить ссылку для входа"
4. Проверьте Supabase Dashboard:
   - Authentication → Users → должен появиться новый пользователь
   - Или проверьте почту (если настроен SMTP)

### 4. Получение magic link

**Вариант A: Через Supabase Dashboard**
1. Authentication → Users → найдите пользователя
2. Скопируйте confirmation token из URL
3. Вставьте в: `http://localhost:3000/auth/callback?code=TOKEN`

**Вариант B: Через Inbucket (если настроен)**
1. Откройте http://localhost:54324 (Supabase Inbucket)
2. Найдите письмо
3. Кликните по ссылке

**Вариант C: Через реальную почту (если SMTP настроен)**
1. Проверьте почту
2. Кликните по ссылке в письме

### 5. Проверка авторизованного состояния

После успешного входа:

- ✅ Редирект на главную страницу
- ✅ В Header отображается баланс (0.00 ₽)
- ✅ В Header отображается статус (Bronze)
- ✅ Кнопки "Профиль" и "Выход" вместо "Войти"

### 6. Проверка профиля

1. Нажмите "Профиль" в Header
2. Проверьте отображение:
   - ✅ Email пользователя
   - ✅ Статус Bronze
   - ✅ Баланс 0.00 ₽
   - ✅ Реферальный код (8 символов)
   - ✅ Дата регистрации
   - ✅ Кнопка "Копировать" для реферальной ссылки

### 7. Проверка middleware

**Тест 1: Доступ к защищённому роуту без авторизации**
1. Выйдите из системы (кнопка "Выход")
2. Попробуйте открыть http://localhost:3000/profile
3. ✅ Должен произойти редирект на /auth/login?redirect=/profile

**Тест 2: Доступ к защищённому роуту с авторизацией**
1. Войдите в систему
2. Откройте http://localhost:3000/profile
3. ✅ Страница профиля должна открыться

### 8. Проверка выхода

1. Нажмите "Выход" в Header
2. ✅ Редирект на /auth/login
3. ✅ В Header снова кнопка "Войти"
4. ✅ Попытка открыть /profile редиректит на /auth/login

### 9. Проверка API

**GET /api/user/profile (авторизован)**
```bash
curl http://localhost:3000/api/user/profile \
  -H "Cookie: sb-access-token=YOUR_TOKEN"
```
✅ Должен вернуть профиль пользователя

**GET /api/user/profile (не авторизован)**
```bash
curl http://localhost:3000/api/user/profile
```
✅ Должен вернуть 401 Unauthorized

### 10. Проверка базы данных

Откройте Supabase Dashboard → Table Editor → users:

- ✅ Создана запись с вашим email
- ✅ balance = 0
- ✅ status_id указывает на Bronze
- ✅ referral_code заполнен (8 символов)
- ✅ created_at заполнен

## Известные ограничения

1. **Email не приходит локально** — используйте Supabase Dashboard для получения токена
2. **Поиск в Header** — пока заглушка, будет реализован в Этапе 3
3. **Кнопка "Пополнить баланс"** — пока не работает, будет в Этапе 4
4. **Админ-панель** — будет в Этапе 7

## Troubleshooting

### Ошибка "Invalid API key"
- Проверьте `.env.local` — ключи должны быть корректными
- Перезапустите dev-сервер после изменения .env

### Ошибка "relation users does not exist"
- Примените схему БД: `supabase_schema.sql`
- Проверьте подключение к правильному проекту

### Magic link не работает
- Проверьте Site URL в Supabase Dashboard → Authentication → URL Configuration
- Должен быть: http://localhost:3000

### Middleware не редиректит
- Проверьте, что middleware.ts находится в src/
- Перезапустите dev-сервер
